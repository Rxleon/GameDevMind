# 静态站点方案评估

> 评估日期：2026-06-20  
> 现状：GitHub Pages 托管 `nav/` 交互式导航 + 仓库内 Markdown 直读

## 需求

| 需求 | 权重 |
|------|------|
| 中文文档 100+ 篇可读性 | 高 |
| 搜索（全文 / 关键词） | 高 |
| 与现有 `mds/` 结构兼容 | 高 |
| 维护成本低 | 高 |
| 自定义交互导航保留 | 中 |

## 方案对比

| 方案 | 优点 | 缺点 | 建议 |
|------|------|------|------|
| **现状（GitHub + nav/）** | 零迁移、PR 即更新 | 全文搜索弱、样式不统一 | ✅ 短期保留 |
| **MkDocs + Material** | Markdown 原生、中文友好、内置搜索 | 需维护 `mkdocs.yml` 导航树 | ⭐ 首选候选 |
| **VitePress** | Vue 生态、速度快、主题现代 | 对 HTML 混排兼容需验证 | 备选 |
| **Docusaurus** | 版本化、博客、i18n | 偏重、React 栈 | 多语言成熟后再考虑 |

## 推荐路径

**阶段 1（当前）**

- 继续 GitHub Pages + [nav/index.html](../nav/index.html)
- 使用自动生成的 [INDEX.md](../INDEX.md)、[KEYWORDS.md](../KEYWORDS.md)
- 阅读路径见 [推荐阅读路径.md](../mds/topics/推荐阅读路径.md)

**阶段 2（可选）**

- 试点 MkDocs：仅导入 `mds/1.基础能力/` 一个模块
- 评估 HTML 标签、表格、图片路径渲染效果
- 通过 GitHub Actions 部署到 `gh-pages` 子路径（如 `/docs/`）

**阶段 3（若试点通过）**

- 全量迁移或双轨发布（仓库 Markdown + 站点镜像）
- 将 `nav/` 三条路径筛选逻辑迁入 MkDocs 插件或自定义 JS

## 结论

**暂不整体迁移。** 当前 `nav/` + INDEX + KEYWORDS 已覆盖主要导航需求；若后续搜索反馈增多，优先试点 **MkDocs Material**，而非一次性替换 Docusaurus。

## CDN / 懒加载

图谱大图在 `exports/`，体积较大。短期：

- 使用 [check_images.py](../tools/check/check_images.py) 审计大文件
- PNG 优化脚本 `--compress`（需 Pillow）

长期若 Pages 加载慢，可将 `exports/` 迁至 CDN，文档内改为绝对 URL（需批量脚本 + CONTRIBUTING 约定）。

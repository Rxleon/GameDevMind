## 🎯 待优化项

### 1. 文档格式规范统一化

**优先级：高**

**问题**：`CONTRIBUTING.md` 要求纯 Markdown（不使用 HTML 标签），但大量文档仍使用 `<h2 align="center">` 等 HTML 样式，规范与实际不一致。

**建议**（二选一）：

- **方案 A（推荐）**：更新 `CONTRIBUTING.md`，明确允许 HTML 标签用于标题居中、表格样式等，并同步 `format-check` workflow
- **方案 B**：批量将文档改为纯 Markdown（工作量大）

**关联**：`template/z.模板.md` 仍含旧格式与 `?raw=true`，需与最终规范对齐后完善模板。

### 4. 内容质量（剩余）

#### 4.1 扩展 CI / 本地检查

**优先级：高**

已有链接与基础格式检查（见 [.github/workflows/README.md](.github/workflows/README.md)），可继续扩展：

- 必填字段检查（关键词、标签等）
- 图片存在性检查（引用路径 vs 实际文件）
- 内容完整性检查（「是什么？在哪用？」等结构）
- 提供本地可运行的检查脚本，方便贡献者提交前自检

#### 4.2 配套资源持续扩充

**优先级：低**

已有 10 + 8 + 5 配套资源，可按图谱覆盖度继续补：

- 更多文档与 code / cases / ai-cases 的映射
- 技术类文档中的内联代码片段与反例

#### 4.3 内容审核

**优先级：低**

- 内容审核清单
- 文档质量评分标准
- 定期审查与更新机制

## 📋 实施优先级

### 🔴 高优先级

1. **明确文档格式规范**：统一 CONTRIBUTING、模板、format-check
2. **扩展文档检查**：图片存在性、必填字段、本地检查脚本

### 🟡 中优先级

1. **README 补充**：INDEX.md、HISTORY.md 链接
2. **图片优化**：压缩与缺失检测脚本
3. **命名规范文档**

### 🟢 低优先级

1. GitHub 搜索说明 / 关键词索引
2. 静态站点生成器评估
3. 工具统一配置、元数据、CDN、FAQ 等
